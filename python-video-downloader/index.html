Python Video Test Downloader | A failure

This was one of my first real projects. A "simple" python application for video downloading, becoming a failure. The core idea was simple enough but the execution collapsed under its own weight. The threaded build produced video files that were technically valid but unplayable in any standard media player, and rather than diagnose why, I just kept adding layers around the problem. The real irony is that the "standalone" executable wasn't standalone at all. It relied on 7-zip being pre-installed or installed on launch to unpack its own bundled dependencies at runtime, meaning self-contained, still silently assumed two external tools on the user's machine before it could do anything. This resulted in my learning curve towards C# and the app I built with that change of mindset.

Technical Challenges

01. Ignorance

As a beginner in software development, I faced a steep learning curve in understanding the fundamentals of programming, software architecture, and best practices. My lack of experience led to poor design decisions, such as tightly coupling components and neglecting error handling, which ultimately contributed to the project's failure. This experience taught me the importance of foundational knowledge and the need to continuously learn and improve my skills as an engineer.

02. Overcomplexity

In an attempt to add features and functionality, I overcomplicated the codebase with unnecessary layers of abstraction and convoluted logic. This made the code difficult to understand, maintain, and debug, leading to a fragile application that was prone to breaking with any changes. This experience highlighted the importance of simplicity and clarity in software design, and the need to prioritize maintainability over feature bloat.

03. Dependencies

The project relied heavily on external dependencies, such as 7-zip and ffmpeg, without properly managing them or ensuring they were included in the final executable. This resulted in a non-functional application that attempted to fix itself with my own code, introducing unneeded environment variables, dependency installations and crashes. This experience taught me the importance of proper dependency management and the need to ensure that all necessary components are included and properly configured in a software project.
