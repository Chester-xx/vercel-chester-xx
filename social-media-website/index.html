Technical Challenges

01. Moving Parts

Managing the various components of the application, including user authentication, database interactions, and frontend design, was a significant challenge. Ensuring that all parts worked together seamlessly required careful planning and debugging. I had to learn how to structure the codebase effectively, manage state across different pages, and handle user sessions securely. This experience taught me the importance of modular design and clear separation of concerns in web development.

02. Security

Implementing secure user authentication and protecting against common vulnerabilities like SQL injection and cross-site scripting (XSS) was a critical challenge. I had to learn about hashing passwords, validating user input, and implementing secure session management. This experience highlighted the importance of security best practices in web development and the need to stay informed about potential threats and mitigation strategies.

03. Time Constraints

Balancing the scope of the project with the time available was a constant challenge. I had to prioritize features and make trade-offs to ensure that I could deliver a functional product within the deadline. This experience taught me valuable lessons in project management, time estimation, and the importance of iterative development. It also reinforced the need to focus on core functionality before adding additional features.
